🔍 PDF Accessibility Crawler

Submit a website URL to discover all PDF files it contains and assess their accessibility using the checks from simplA11yPDFCrawler.

How it works

  1. Fill in the form below and click Submit Crawl Request.
  2. You are redirected to GitHub to create a SCAN: issue, which triggers the Crawl Site for PDFs workflow (runs up to 1 hour).
  3. Once the crawl is complete, the Analyse PDFs for Accessibility workflow starts automatically and updates reports/report.md.
  4. PDFs are deleted after analysis; only the YAML manifest is kept (with MD5 hashes, so unchanged files are not rescanned).

Submit a Crawl via GitHub Issues

Fill in the URL below and click Open GitHub Issue. You will be taken to GitHub with a pre-filled issue title and body — just click Submit new issue to start the crawl.

Enter the full URL including https://.

Prefer to create the issue yourself? Open a new GitHub issue and set the title to SCAN: https://example.com.

View Reports

  • Markdown report – human-readable summary of all analysed PDFs
  • JSON report – machine-readable full results
  • YAML manifest – complete tracking file with MD5 hashes
